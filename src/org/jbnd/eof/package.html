Classes that connect JBND and Enterprise Objects Frameworks (by Apple TM). EOFDataObject is a DataObject implementation that extends EOGenericRecord, and should be used as a superclass (direct or otherwise) of all client side EO classes in a WOJC application using JBND. EOFDataType is a DataType implementation that encapsulates an EOClassDescription object and translates DataType method calls to EOClassDescription ones. EOFDataProvider should be used to obtain EOFDataType objects. EOFValidationException translates from EOF's NSValidation.ValidationException to org.jbnd.ValidationException.
